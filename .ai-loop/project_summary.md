# Project Summary

## Project Purpose

`ai-orchestrator` is a local PowerShell-based AI development loop that coordinates an implementer (Cursor Agent by default; OpenCode/Qwen via wrapper), Codex review, optional tests, and guarded git commit/push using a file contract under `.ai-loop/`.

## Current Architecture

- `ai_loop.py` -- experimental GitHub PR orchestrator; separate from the primary PowerShell loop.
- `scripts/ai_loop_task_first.ps1` -- task-first entry: auto-refreshes `.ai-loop/repo_map.md` at start when missing or older than one hour (non-fatal); **default-on preflight** parses `## Files in scope` in `task.md` and exits before the implementer when a listed path is missing, not marked with ` (new)` at end of line or immediately after the path as a parenthetical starting with `new` (e.g. `(new)`, `(new, optional)`; same-line notes after it are allowed), and not a glob or directory-style entry (`*`, `?`, or trailing `/`/`\`); use `-SkipScopeCheck` to bypass. **Manual `chain.json`:** after preflight, `Initialize-AiLoopPlannerChain` with manual `planner_form` runs **before** the `-SkipInitialCursor` branch so skip-implementer task-first runs still open or preserve a chain for JSONL accounting. Resets `implementer_summary.md`, saves the effective implementer selection, runs the configured implementer, gates on a path-set delta plus `.ai-loop/implementer_result.md` mtime/existence, then invokes `ai_loop_auto.ps1`. See **DD-022** for optional `-WithScout` and **DD-023** for optional `-WithWrapUp`.
- `scripts/ai_loop_plan.ps1` -- **manual** opt-in task planner: builds a full prompt from `templates/planner_prompt.md` (or installed `.ai-loop/planner_prompt.md`) plus `AGENTS.md`, `project_summary.md`, optional `repo_map.md`, and a user ASK (`-Ask` or `-AskFile`), then invokes `-PlannerCommand` (default `run_claude_planner.ps1`) to draft `.ai-loop/task.md`. After each successful write, if the drafted task contains optional `## Order` with a positive integer on the following line, a duplicate is copied to `tasks/NNN_slug.md` under the orchestrator repo root (`Split-Path $PSScriptRoot -Parent`), creating `tasks/` when missing (non-fatal; failures warn only). Optional `-WithReview` runs up to three Codex review iterations (hard cap enforced even if `-MaxReviewIterations` is raised; values below 1 clamp to 1) via `run_codex_reviewer.ps1` plus `templates/reviewer_prompt.md` or `.ai-loop/reviewer_prompt.md`; reviewer is advisory and the planner has final say (`Architect note:` for rejections); reviewer stdout is mechanically validated (`Test-ReviewerOutputStrict`: trimmed body is exactly `NO_BLOCKING_ISSUES` or an `ISSUES:` block whose non-blank lines are only `- [logic|complexity|scope|missing|architecture|safety] …`; otherwise `REVIEWER_OUTPUT_MALFORMED` degrades without planner revision); trace overwrites `.ai-loop/planner_review_trace.md` with `REVIEW_STATUS:` lines on failed reviewer, malformed output, failed planner revision, and failed revision sanity; the max-iteration exit line is emitted only when the loop actually consumes all iterations (not on early clean exit or degraded break); reviewer/transient failures degrade gracefully by writing the last good draft without failing the exit code except initial planner/setup errors (`exit 1` only for prerequisites or planner wrapper failures on initial draft per C07 semantics; `exit 2` unchanged for initial-draft sanity failures). Worst-case cost when enabled: seven LLM calls (one initial planner + three reviewer + three revision). Not part of the automated loop; architect-swappable like `-CursorCommand`; structural sanity via `Test-PlannerOutputSanity`; human review of `task.md` remains the quality gate.
- Optional `-WithDraft` on `ai_loop_plan.ps1`: before Claude, stdin-runs `-DraftCommand` (default `run_cursor_agent.ps1`; literal path or `.\`-relative paths resolve against the working directory / project root like `-PlannerCommand`, and wrapper names without path separators additionally resolve beside `ai_loop_plan.ps1` in `$PSScriptRoot`) on a prompt built from `.ai-loop/draft_brief_prompt.md` or `templates/draft_brief_prompt.md` plus the same `AGENTS.md` / `project_summary.md` / optional `repo_map` / USER ASK bundle; wrapper failure (non-zero exit or terminating error, try/catch), missing wrapper, or trimmed UTF-8 output under 50 bytes is non-fatal (warnings; no brief). After the draft phase, `LASTEXITCODE` is reset before invoking the planner so a failed draft cannot make the planner pass look unsuccessful. Otherwise overwrites gitignored `.ai-loop/task_draft_brief.md` and appends an advisory `## Cursor Draft Brief` section after `## USER ASK` in Claude's initial planner prompt and planner revision prompts (`-WithReview`).
- `scripts/run_claude_planner.ps1` -- Claude CLI wrapper for the planner (`claude --print`); stdin prompt; no `param()` block; same stderr/`NativeCommandError` workaround pattern as `run_opencode_agent.ps1`.
- `scripts/run_claude_reviewer.ps1` -- Claude Haiku-based architecture reviewer for an opt-in cheap planning mode; use with `-PlannerCommand` set to a Cursor (or other) planner wrapper, `-ReviewerCommand` pointing at this script, `-WithReview`, and `-NoRevision` (variant A only: one review pass, no auto-revision). `Test-ReviewerOutputStrict` accepts `[architecture|safety]` issue tags in addition to the original categories (backward compatible). Reviewer context omits `repo_map.md` when `-NoRevision` is set; blocking `ISSUES:` prevents writing `.ai-loop/task.md` and exits `2`. `templates/claude_task_reviewer_prompt.md` is installed to `.ai-loop/claude_task_reviewer_prompt.md` by `install_into_project.ps1`. On exit code zero, merges stdout/stderr like other Claude wrappers and, when usage is parseable, appends a row to `.ai-loop/token_usage.jsonl` (`ScriptName` distinguishes reviewer from planner passes).
- `scripts/run_codex_reviewer.ps1` -- Codex CLI wrapper for optional planner advisory review (`codex exec` positional prompt bridged from stdin); no script-level `param()` block; duplicated `ConvertTo-CrtSafeArg`; same `$ErrorActionPreference` NativeCommand workaround as `run_claude_planner.ps1`.
- `scripts/ai_loop_auto.ps1` — test/diff capture, Codex review, fix-loop, final test gate, safe staging, commit/push. `Save-TestAndDiff` / `Save-GitReviewArtifactsForCodex` write scope-filtered **`git_status.txt`**, **`diff_summary.txt`**, and **`last_diff.patch`** for Codex (DD-024: DurableAlwaysCommit plus ActiveScope entries under SafeAddPaths; bare `git diff HEAD` only when that path list is empty). `Stage-SafeProjectFiles` stages DurableAlwaysCommit when present. **`tasks/` entries in `SafeAddPaths` log a one-line `[scope-filter]` skip when `## Files in scope` omits tasks** (no blind `git add tasks/`); when scope mentions tasks, paths listed in scope are staged as usual. **ActiveScope file paths that are deleted in the working tree but still in the Git index are `git add`’d** so removals enter the commit; empty scope fails closed with `[scope-filter]` warning. **C12:** after `Extract-FixPromptFromFile`, paths under `tasks/` in the fix prompt are checked via `Test-FixPromptArtifactsTasksConflict` (JSON `files`/`changes` plus parsed `## Files to change` / `## Changes` in the rendered markdown when JSON is absent); `Stop-UnsafeQueueCleanup` exits before the implementer on conflict. The same conflict gate runs when `-Resume` reloads an existing `next_implementer_prompt.md` before `Run-ImplementerFix`. **`Commit-And-Push`** (Codex PASS / final gate) no longer runs a working-tree `tasks/` pre-gate for parallel queue files; protection stays on the fix-prompt path above. **Resume (`-Resume`):** when `Try-ResumeFromExistingReview` invokes `Run-ImplementerFix -FixIterationIndex 1` before the outer loop, `$script:AiaResumePreloopFixIterations` offsets **`Run-CodexReview`** `fix_iteration_index` (joined capture → JSONL) and loop **`Run-ImplementerFix`** indices so numbering matches uninterrupted runs. Codex instructions use a single-quoted here-string so fenced JSON survives for `codex exec` and `Extract-FixPromptFromFile`. Optional `-WithWrapUp` (DD-023) runs `scripts/wrap_up_session.ps1` post-pass toward `.ai-loop/_debug/session_draft.md` without affecting exit codes when wrap-up errs; it also **appends a non-blocking token JSONL row** (`phase`/`role` wrap_up; optional `planner_chain_id` / `task_name` from `chain.json` when present).
- `scripts/continue_ai_loop.ps1` -- resume wrapper for `ai_loop_auto.ps1 -Resume`; forwards explicit `-CursorCommand`, `-CursorModel`, and `-WithWrapUp`.
- `scripts/run_cursor_agent.ps1` and `scripts/run_opencode_agent.ps1` -- implementer wrappers; parameter names remain `-CursorCommand` / `-CursorModel` for compatibility.
- `scripts/run_opencode_scout.ps1` -- OpenCode scout role wrapper (same stdin/`opencode run` flow as `run_opencode_agent.ps1` but SCOUT framing so scout instructions are not overridden by the implementer message).
- `scripts/run_scout_pass.ps1` -- optional read-only scout pre-pass for `-WithScout` (DD-022); writes gitignored `.ai-loop/_debug/scout_*` artifacts only. When `-CommandName` points at `run_opencode_agent.ps1`, the pass auto-substitutes `run_opencode_scout.ps1` beside this script to avoid role conflict; warns and falls back if the scout wrapper is missing. Short scout output (under 200 bytes) is treated as a non-fatal startup failure with a warning.
- `tests/test_orchestrator_validation.py` — parser smoke tests, safe-path parity, task-first delta semantics, implementer-state resume behavior, prompt parsing (C02 uses a dot-sourced PowerShell harness so `$STABLE_PREAMBLE` / `Get-TaskScopeBlocks` are not reimplemented in Python), task-first preflight subprocess tests (`-SkipInitialCursor`, fake `-AutoLoopScript`, scratch dir under `tests/` when pytest temp is not writable on Windows), dynamic step-label checks, repo map determinism checks, embedded Codex review prompt context checks (`reviewer_context.md` ordering vs `AGENTS.md`), subprocess coverage for `ai_loop_plan.ps1 -WithDraft` when the draft wrapper throws (planner still runs). Wrap-up (`wrap_up_session.ps1`) plus manual promote (`promote_session.ps1`) checks for DD-023. **`chain.json` / `-ForceNewChain` subprocess coverage lives in `tests/test_token_usage.py`** (scoped per Token ledger v2 task).
- `.ai-loop/repo_map.md` is a committed, script-generated file index. `ai_loop_task_first.ps1` invokes `scripts/build_repo_map.ps1` automatically when the map is absent or stale (>1 h); regenerate manually after structural changes if you need immediate freshness; deterministic output is pinned by tests.
- `templates/` -- target-project scaffolds, including `implementer_summary_template.md`, `codex_review_prompt.md`, `planner_prompt.md`, `draft_brief_prompt.md` (Cursor read-only planner pre-pass wording for `ai_loop_plan.ps1 -WithDraft`; installed to `.ai-loop/draft_brief_prompt.md`; local override beats templates), `reviewer_prompt.md` (narrow advisory planner reviewer text; unrelated to implementation `codex_review_prompt.md`), `claude_task_reviewer_prompt.md` (Claude architecture reviewer prompt for `run_claude_reviewer.ps1`; installed beside `reviewer_prompt.md`), `user_ask_template.md` (hint template copied to `.ai-loop/user_ask_template.md` by the installer, not to `user_ask.md`, which users create or copy manually), `project_summary.md`, `task.md`, and `opencode.json`.
- `scripts/install_into_project.ps1` -- copies orchestrator scripts (including `ai_loop_plan.ps1`, `run_claude_planner.ps1`, `run_codex_reviewer.ps1`, `run_scout_pass.ps1` and `run_opencode_scout.ps1` for DD-022 `-WithScout`, plus `wrap_up_session.ps1` and `promote_session.ps1` for DD-023) and planner/templates (`planner_prompt.md`, `draft_brief_prompt.md`, `reviewer_prompt.md`, `claude_task_reviewer_prompt.md`, `user_ask_template.md`) into a target project; refuses self-install when source and target are the same directory; when adding or removing files under `templates/`, update this script so installs stay complete (see AGENTS.md templates contract).

## Current Pipeline / Workflow

For a new task, run `ai_loop_task_first.ps1`. Optionally draft or refine `.ai-loop/task.md` first with `scripts/ai_loop_plan.ps1` (manual; requires a configured planner CLI). It runs the implementer first, then hands off to `ai_loop_auto.ps1` only after meaningful implementation side effects or an accepted no-code marker in `.ai-loop/implementer_result.md`. For existing changes, run `ai_loop_auto.ps1` directly. Codex reviews after tests/diff capture; `PASS` triggers final tests and safe commit/push unless `-NoPush`; `FIX_REQUIRED` writes `.ai-loop/next_implementer_prompt.md` and reruns the selected implementer.

After each Codex review, `scripts/ai_loop_auto.ps1` prints a short console summary (`PASS` or `FIX_REQUIRED`, optional first-bullet reason on `FIX_REQUIRED`, tokens line when parseable, preferring the trailing Codex `tokens used` footer over embedded API-shaped JSON in the transcript); full transcript and extractor rules remain in `.ai-loop/codex_review.md` and PowerShell helpers in `scripts/ai_loop_auto.ps1`.

Resume uses `.ai-loop/implementer.json` (runtime, gitignored) to reload the last effective wrapper/model when `-CursorCommand` is omitted. Explicit `-CursorCommand` / `-CursorModel` on `continue_ai_loop.ps1` or `ai_loop_auto.ps1 -Resume` override persisted state.

## Important Design Decisions

- Implementer prompt = `$STABLE_PREAMBLE` + `FILES IN SCOPE:` / `FILES OUT OF SCOPE:` blocks parsed from `task.md` + `TASK:` body. Required sections in `templates/task.md`; missing sections produce a warning, not a failure. Task-first also runs deterministic preflight on `## Files in scope` bullet paths (skips globs or directory-like entries and ` (new)` at line end or right after the path as `(new...)` (parenthetical whose first word is `new`)) before the implementer; omitting that section logs a warning and continues. Complements C07’s planner sanity check by catching invented or mistyped paths at launch regardless of how `task.md` was authored. `-SkipScopeCheck` opts out when needed.
- Optional `-WithScout` (DD-022) runs `scripts/run_scout_pass.ps1` before the implementer pass and inserts `RELEVANT FILES (from scout):` after the scope blocks when scout returns a non-empty `relevant_files` list. Off by default; failures are non-fatal.
- Active PowerShell artifacts are implementer-neutral: `implementer_summary.md`, `next_implementer_prompt.md`, `implementer_result.md`, `FIX_PROMPT_FOR_IMPLEMENTER`, and `_debug/implementer_*`. Optional wrap-up emits `_debug/session_draft.md`; running `promote_session.ps1` elevates drafts into tracked `.ai-loop/failures.md` (manual only, rolling retention per DD-023).
- Wrap-up draft headings and the `failures.md` seed use Unicode em dash (U+2014) in emitted markdown. Orchestrator `.ps1` sources build that character with `[char]0x2014` / subexpressions so literals stay ASCII-friendly for Windows PowerShell 5.1 `Parser::ParseFile`.
- Codex emits `FIX_PROMPT_FOR_IMPLEMENTER` as JSON (`fix_required`, `files`, `changes[]`, `acceptance`); `Extract-FixPromptFromFile` prefers JSON and falls back to the legacy free-text regex with a warning. Codex prompt forbids full-suite re-runs (orchestrator pre-captures pytest output); targeted single-test runs are allowed with a one-line reason in `FINAL_NOTE`.
- Legacy Cursor-named alias artifacts are removed from the active PowerShell contract; summary, next-fix prompt, fix label, result marker, and debug outputs use implementer-neutral names.
- `run_cursor_agent.ps1` remains because it is the real Cursor wrapper, not a legacy alias. `-CursorCommand` / `-CursorModel` remain as compatibility parameter names.
- `SafeAddPaths` is the upper bound of what may be staged. **`ai_loop_auto.ps1` (DD-024)** stages **DurableAlwaysCommit** (the `.ai-loop/` paths enumerated in `SafeAddPaths`: task and companion state files, `archive/rolls/`, optional `.ai-loop/_debug/session_draft.md`) on every pass; other paths under `SafeAddPaths` are staged only when listed under `## Files in scope` in `.ai-loop/task.md` (so `tasks/` or `scripts/` blobs are not bulk-staged unless scope says so). Queued planner specs under `tasks/` remain eligible once explicitly in scope.
- **C12 Task queue protection:** Queued specs under `tasks/` are protected in `AGENTS.md` and `templates/codex_review_prompt.md` from deletion or modification recommendations unless the active `.ai-loop/task.md` explicitly includes `tasks/` or the specific file in `## Files in scope` (and requests cleanup when applicable). `SafeAddPaths` still enforces what can be staged; the review-layer wording keeps Codex from treating the queue as disposable scratch. **`Test-FixPromptArtifactsTasksConflict`** / **`Stop-UnsafeQueueCleanup`** in `ai_loop_auto.ps1` enforce this before the implementer: extracted fix prompts (JSON plus markdown fallback) must not reference `tasks/` when scope omits `tasks/`; **resume** re-checks before `Run-ImplementerFix`. Parallel untracked `tasks/*.md` files no longer block **`Commit-And-Push`**; **`Stage-SafeProjectFiles`** skips bulk staging of the `tasks/` `SafeAddPaths` entry when scope omits tasks.
- **C13 Codex review context reduction:** Replaced `AGENTS.md` as default reviewer reading with bounded `templates/reviewer_context.md` (≤40 lines, reviewer-scoped rules only). `test_output.txt` is now conditional on `test_failures_summary.md` being absent or insufficient. `AGENTS.md` remains an explicit fallback (item 9 in the reading list).
- Runtime outputs (`codex_review.md`, diffs, test logs, `implementer_result.md`, `implementer.json`, `_debug/`, final status) are gitignored and not staged by default.
- `docs/architecture.md` is the source of truth for target design; `docs/decisions.md` is the numbered index.

## Known Risks / Constraints

- External CLIs (`codex`, Cursor Agent, OpenCode, local llama.cpp servers) must be installed and authenticated/configured where used.
- Python static analysis uses `pyrightconfig.json`; run `pyright` for Python-only type checks. LSP command: `pyright-langserver --stdio`.
- On Windows PowerShell, empty array returns can become `$null`; path-set helpers emit through the pipeline and callers wrap with `@(...)`.
- `ai_loop.py` still carries older experimental Cursor-centric terminology and is intentionally separate from the active PowerShell loop unless a task explicitly authorizes changing it.
- Under **pwsh**, `Join-Path $base '.ai-loop\…'` can return `$null` when the child path starts with `.`; `scripts/record_token_usage.ps1` and `scripts/show_token_report.ps1` build `.ai-loop` artifact paths with `[System.IO.Path]::Combine` instead.
- Windows PowerShell 5.1 `Parser::ParseFile` on UTF-8 scripts without a BOM can misread multi-byte characters (for example U+2014 in double-quoted lines). Orchestrator `scripts/ai_loop_auto.ps1` avoids literal em dash bytes in source; use `$([char]0x2014)` when scripts must *emit* that character (see `wrap_up_session.ps1`).
